## 构建目标
- 使用 Taro 的 weapp 构建脚本重新打包，生成完整可导入的微信小程序产物到 `dist/weapp`。

## 前置检查
- 包管理器与脚本：确认存在 `package.json` 的脚本 `build:weapp`。
- 框架与配置：项目为 Taro（Vue3），输出目录为 `dist/weapp`（`project.config.json` / Taro 配置）。

## 打包步骤
- 开发模式打包：在项目根目录执行 `pnpm run build:weapp`。
- 生产模式体积优化（可选，Windows PowerShell）：
  - ` $env:NODE_ENV='production'; pnpm run build:weapp `
  - 完成后可清空：` $env:NODE_ENV=$null `

## 产物验证
- 确认生成目录：`dist/weapp`。
- 关键文件：`app.json`、`project.config.json`、`pages/*`、`runtime.js`、`vendors.js`。

## 开发者工具导入
- 打开微信开发者工具 → 导入项目。
- 目录选择：`dist/weapp`。
- 填写小程序 `AppID`，即可本地预览与上传。

## 构建警告处理（不阻塞本次打包）
- Sass `@import` 弃用警告：后续将样式迁移到 `@use` / `@forward`。
- mini-css-extract-plugin 样式顺序冲突：后续检查依赖样式的引入顺序或拆分页面样式，减少交叉依赖。
