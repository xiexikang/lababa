## 目标
- 将“一起拉”从计时区按钮组中移除，改为首页右侧悬浮按钮，始终可见
- 在微信小程序端点击即可打开分享面板；H5 端点击给出分享提示
- 样式与现有卡通风格统一，文字不换行

## 变更点
1. 页面结构（`src/pages/index/index.vue`）
- 删除按钮组中的“一起拉”按钮，仅保留“拉完了”“请祈祷”两枚
- 在根容器（`Index-root`）内新增悬浮按钮区：
```vue
<!-- 右侧悬浮分享 -->
<view class="floating-share">
  <!-- weapp 端：原生分享按钮 -->
  <button v-if="env === 'WEAPP'" open-type="share" class="share-inner">
    👥 一起拉
  </button>
  <!-- 其他端：点击触发提示 -->
  <nut-button v-else class="share-inner" color="#ff69b4" @click="dataInfo.shareTogether()">
    👥 一起拉
  </nut-button>
</view>
```
- 在 `<script setup>` 中新增运行环境判断：
```ts
import Taro from '@tarojs/taro'
const env = Taro.getEnv() // WEAPP / H5 / RN 等
```

2. 逻辑（`index.vue`）
- 保留已配置的 `onShareAppMessage`、`onShareTimeline`，并继续在 H5 端使用 `dataInfo.shareTogether()` 做提示（不再尝试打开分享面板）
- 逻辑不影响现有计时/保存流程

3. 页面配置（`src/pages/index/index.config.ts`）
- 已开启：`enableShareAppMessage: true`、`enableShareTimeline: true`（无需变更）

4. 样式（`src/pages/index/styles/Index.scss`）
- 新增悬浮样式，右侧居中靠下，保持不换行、丝带风格：

- 将原有 `.btn-group` 选择器里的尺寸规则同步给 `.share-btn`（如需），确保三端视觉一致

## 验证
- H5：按钮固定在右下，点击提示“请在微信小程序中分享”；两枚主按钮不换行
- 微信小程序：右侧悬浮按钮为原生 `open-type="share"`，点击打开分享面板；页面右上角分享也可用
