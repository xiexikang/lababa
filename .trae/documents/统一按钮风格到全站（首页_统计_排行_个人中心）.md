**目标**

* 统一按钮圆角、配色、阴影与尺寸到全站，兼容小程序原生 `button` 与 H5 的 `nut-button`。

**方案**

* 新增全局样式文件：`src/styles/components/_buttons.scss`

  * 基础类：`.btn`（布局、圆角、尺寸、居中、字重、对比色、阴影、active 态缩放）

  * 主题类：`.btn--primary(#8BCE92)`、`.btn--warning(#ffb60d)`、`.btn--danger(#ff6b6b)`、`.btn--pink(#ff69b4)`

  * 尺寸类：`.btn--lg(320×120rpx)`、`.btn--md(260×96rpx)`、`.btn--sm(200×80rpx)`

  * 线框类：`.btn--outline`（白边、透明背景、文字深绿）

  * 禁用态：`.is-disabled`（降低对比、屏蔽点击）

* 在页面样式中引入：`@use '@/styles/components/buttons' as *;` 或 `@import`（保持现版本兼容）

* 统一按钮结构：

  * 小程序端使用原生 `button.btn.btn--primary.btn--lg` 等；文案直接写在标签内，不嵌套 `<text>`

  * H5 端使用 `nut-button` 并附加相同类名：`class="btn btn--primary btn--lg"`

* 提升可见性：统一设置 `z-index:10`，避免被弹窗/遮罩覆盖；必要时页面局部调高容器层级。

**页面落地**

* 首页（start/complete/help/分享悬浮）

  * `start`：`.btn btn--primary btn--lg`

  * `complete`：`.btn btn--outline btn--md`

  * `help`：`.btn btn--warning btn--md`

  * 分享悬浮：`.btn btn--pink btn--sm` 固定定位

* 统计页

  * 弹窗按钮与列表操作统一 `.btn` 系列类

* 排行页

  * “加载更多”“邀请粑友”等统一 `.btn` 类

* 个人中心

  * 授权登录按钮统一 `.btn btn--primary btn--md`

**兼容注意**

* 保持现有页面的局部色系不变（绿/黄/粉/红），只通过统一类管理

* 小程序端避免属性选择器；统一用类控制样式

* 禁用态通过附加 `.is-disabled` 或 `disabled` 属性（小程序原生/`nut-button` 均支持）

**实施步骤**

1. 新增 `_buttons.scss` 并引入到各页面样式（或在 `custom_theme.scss` 中集中引入）
2. 替换页面按钮类名为 `.btn + 主题 + 尺寸`，移除内层 `<text>`
3. 验证在微信开发者工具与 H5：展示、点击态、禁用态与层级
4. 重新打包：`npm run build:weapp`；可选启动 `npm run dev:weapp` 实时查看

**验收**

* 四个页面按钮外观一致（圆角、色值、阴影、尺寸）

* 小程序端按钮始终可见、可点击；H5 端保留 `nut-button` 行为同时风格统一

* 禁用态、active 缩放
