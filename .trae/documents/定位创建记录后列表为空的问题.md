## 重点核查点
- 猫归属：用 `GET http://localhost:8081/api/cats/detail/<catId>` 验证该猫存在且 `userId` 等于当前登录用户。
- 鉴权头：确认创建请求包含 `Authorization: Bearer <token>`，与当前列表请求使用同一 token。
- 响应与错误码：记录创建接口返回的 `HTTP 状态`、`code` 与 `msg`（后端已细化 400/404/403）。
- DB 一致性：确认你连接的 MySQL 与 `server-go/.env` 的 `MYSQL_DSN` 指向同一实例与库。

## 复现场景（不改代码）
1. `GET /api/cats/detail/<catId>` 检查猫与用户归属；非 200 则创建必失败。
2. `POST /api/records/create` 使用相同 `Authorization` 与上述 `catId`，最小体例：`{"catId":"<id>","duration":60}`。
3. `POST /api/records/list` 不带筛选，期望看到新记录；若无，马上在 MySQL 执行 `SELECT id,userId,catId,createdAt FROM lababa.records ORDER BY createdAt DESC LIMIT 10;`。
