**目标**

* 打包当前前端为微信小程序产物并在微信开发者工具运行，保持 H5 不受影响。

**执行步骤**

* 校验入口与配置：

  * 确认 `src/app.config.ts` 的 `entryPagePath` 指向 `pages/index/index`（或你指定的入口）。

  * 不改动 H5 路由与 dev server。

* 构建产物：

  * 执行一次性打包命令：`npm run build:weapp`，输出目录为 `dist/`。

* 在微信开发者工具运行：

  * 导入项目目录 `dist`（不是 `src`），设置你的 `AppID`。

  * 点击“编译”，如未看到最新改动，使用“清缓存并重启”后再次编译。

* 可选：开启实时编译（不影响 H5）：

  * 运行 `npm run dev:weapp` 持续监控增量输出到 `dist`；在微信开发者工具中勾选“监视文件变更”“自动编译”。

**验证要点**

* 首页、统计页、个人中心样式与弹窗呈现正确；按钮在小程序端使用原生控件或统一类样式已兼容。

* 请求层统一鉴权已生效（公开接口无需 token，其余自动带 `Authorization` 并处理 401）。
