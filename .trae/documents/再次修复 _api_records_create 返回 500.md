## 变更目标
- 移除插入后的二次查询，直接返回刚构造的记录对象，避免查询失败触发 500。
- 在插入失败和受影响行数为 0 时，记录详细错误日志，以便定位。
- 排行榜更新保持非阻断：失败不影响主记录创建。

## 具体修改（server-go/main.go）
- 替换 `row := db.QueryRow(...)` 与随后 `row.Scan(...)` 的整段为直接构造 `Record` 并返回。
- 在 `if err != nil` 和 `RowsAffected()==0` 分支增加 `log.Printf` 打印 `userID/catId/err`。

## 执行与验证
1. 更新代码、重建并重启 `server-go`（监听 `8081`）。
2. 使用你提供的示例负载再次调用创建接口，期望 `code=0` 返回记录。
3. 调用 `/api/records/list` 确认记录出现；数据库检查 `lababa.records` 有新行。
4. 若仍失败，查看 `run-8081.log` 的错误日志以定位数据库具体异常。
